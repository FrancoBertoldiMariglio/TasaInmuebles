# RF-017 — Pantalla de bienvenida con CTA bifurcado

## Descripción

Al abrir la app mobile **sin sesión activa**, el sistema muestra una pantalla de bienvenida con dos CTAs grandes:

- **"Soy Tasador (arquitecto matriculado)"** → flujo CU-UI-002 (login profesional).
- **"Soy Cliente / Quiero tasar mi propiedad"** → flujo CU-UI-014 (autoregistro Cliente B2C, RF-018).

## Flujo

1. Usuario abre la app.
2. Sistema verifica si hay token JWT válido en storage local.
3. **Si hay token válido**: redirige según `role`:
   - `role = tasador` → [[T-033]] "Mis tasaciones" (CU-UI-003).
   - `role = cliente_b2c` → "Mis tasaciones referenciales" (home del cliente B2C).
4. **Si no hay token o expiró**: muestra pantalla de bienvenida con los 2 CTAs.

## UI

- Logo Tasa Inmuebles arriba.
- Tagline corto.
- 2 botones grandes (uno arriba del otro, no lado a lado, para touch fácil).
- Botón "Soy Tasador" — estilo principal (color primario).
- Botón "Soy Cliente" — estilo secundario (color acento).
- Link discreto "Términos y Privacidad" al pie.

## Errores

- E-001: Token JWT corrupto o inválido → tratar como sin sesión, mostrar pantalla.
- E-002: Sin conexión → permitir mostrar la pantalla (es estática), pero los flujos posteriores requieren conexión.

## Fuera del MVP-6sem
- Login social (Google, Apple).
- Magic links.
- "Recordarme" / sesión persistente con refresh tokens.
- Onboarding tutorial primera vez.
