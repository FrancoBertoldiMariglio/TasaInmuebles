# RF-008 — Validar y guardar tasación

## Descripción
El sistema valida los campos del draft y lo persiste como [[T-026]] en estado [[T-014]] **Inspección hecha** cuando el tasador confirma "Guardar cambios".

## Flujo
1. Tasador toca botón "Guardar cambios".
2. Sistema corre **validaciones**:
   - `tipo_inmueble` ∈ catálogo (RF-001).
   - `motivo` ∈ catálogo (RF-002).
   - `ubicacion`: al menos `(lat, lng)` o `domicilio` no vacío (RF-003).
   - `solicitante`: nombre, apellido, teléfono completos (RF-004).
   - `detalles`: superficies, antigüedad, estado_conservacion completos; dormitorios/baños si aplica (RF-005).
   - `fotos.length` ≥ 1 y ≤ 15 (RF-006).
   - `descripcion.length` ≥ 50 (RF-007).
3. **Si falla validación:**
   - Sistema muestra mensaje específico ("Faltan: ubicación, ≥ 1 foto, descripción muy corta").
   - Sistema resalta secciones con error.
   - Mantiene el draft en pantalla.
4. **Si pasa validación:**
   - Sistema persiste la tasación en BD con estado `Inspección hecha`.
   - **Sistema dispara RF-016** (cálculo automático del valor técnico Fitt-Servini) — el resultado queda persistido en la tasación.
   - Sistema dispara transición automática a `Por tasar` (RF-009).
   - Sistema muestra confirmación + redirige a [[T-033]] "Mis tasaciones".

## Auto-save (recomendado MVP)
- Cada N segundos o al cambiar de acordeón, sistema persiste el estado actual del draft como [[T-002]] Borrador. Si el tasador pierde conexión o cierra la app, los datos quedan recuperables.

## Validación post-guardado
- ID generado correctamente.
- `fecha_alta` = timestamp del guardado.
- `tasador_id` = ID del tasador autenticado.
