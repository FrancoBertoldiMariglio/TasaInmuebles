# RF-018 — Autoregistro de Cliente B2C (email + password)

## Descripción

El sistema permite a un [[T-037]] **Cliente B2C** autoregistrarse desde la app mobile con email + password sin verificación por email (simplificado para MVP).

## Flujo

1. Cliente B2C toca "Soy Cliente / Quiero tasar mi propiedad" en RF-017.
2. Sistema muestra formulario:
   - Email (input email, requerido, validación formato RFC).
   - Password (input password, requerido, min 8 caracteres).
   - Confirmar password (debe coincidir).
   - Nombre (opcional MVP, requerido Fase 2 para PDF y mail).
   - Checkbox "Acepto términos y consentimiento de datos personales (Ley 25.326)" — **requerido**.
3. Cliente B2C completa y toca "Registrarme".
4. Sistema valida:
   - Email no existe ya en BD.
   - Password ≥ 8 caracteres (hashed bcrypt o argon2 antes de persistir).
   - Checkbox marcado.
5. Sistema crea registro `user` con `role = cliente_b2c` + `email_verified = false`.
6. Sistema genera JWT con `sub = user_id` + `role = cliente_b2c` + expiración 8h (igual que tasador).
7. Sistema retorna JWT al cliente.
8. Cliente persiste JWT en secure storage local.
9. Cliente redirige a "Mis tasaciones referenciales" (home Cliente B2C).

## Validación

- Email único en BD.
- Email formato RFC.
- Password mínimo 8 caracteres.
- Hash bcrypt o argon2 (nunca plaintext).
- Checkbox consentimiento marcado **obligatorio** — sin esto el registro falla (compliance Ley 25.326, DS-04).
- JWT con expiración 8h.

## Errores

- E-001: email ya registrado → mensaje "Este email ya tiene cuenta. ¿Querés iniciar sesión?" + link a flow de login.
- E-002: password muy corta → resalta + mensaje "Mínimo 8 caracteres".
- E-003: passwords no coinciden → resalta + mensaje.
- E-004: checkbox no marcado → "Aceptá los términos para continuar."

## Compliance

- Cumple **RG-003 (TLS)**: todo el flujo va por HTTPS.
- Cumple **AC-005 pendiente (Ley 25.326)**: checkbox de consentimiento explícito + log del evento con timestamp.
- En Fase 2: agregar verificación de email + reset de password + cambio de password.

## Login posterior (caso uso parejo, mismo RF base)

Una vez registrado, el Cliente B2C inicia sesión en sesiones futuras con la misma pantalla del CTA bifurcado:
- Toca "Soy Cliente" → si ya hay token válido, salta directo al home.
- Si no, ve un selector "Crear cuenta" / "Ya tengo cuenta" → si elige "Ya tengo cuenta", flujo de login simple (email + password, similar a RF-010 del tasador).

## Fuera del MVP-6sem

- Verificación de email post-registro (envío de link de confirmación).
- Recuperación de password ("Olvidé mi contraseña").
- Cambio de password desde la app.
- 2FA / MFA.
- Login social (Google, Apple).
- Magic links.
- Migración a Auth0 / Clerk (Fase 2 con migración del tasador también).
