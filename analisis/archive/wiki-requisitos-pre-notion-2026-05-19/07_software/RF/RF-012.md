# RF-012 — Generar PDF de tasación

## Descripción
El sistema genera un PDF con los datos de una [[T-026]] en estado [[T-027]] Tasada o posterior.

## Contenido del PDF
- Encabezado: logo Tasa Inmuebles + "Tasación N° {id}".
- Datos del inmueble: domicilio, tipo, motivo, superficies, antigüedad, estado_conservacion, amenities.
- Foto principal (1 grande) + grid de fotos restantes.
- Descripción.
- Datos del solicitante.
- Valor final: ARS y USD.
- Fecha de tasación.
- Participantes del comité (IDs + matrículas).
- Tasador autor (ID + matrícula).
- Firma "Tasa Inmuebles" (texto + sello digital).
- Footer: "Esta tasación es simple, sin inspección ocular del colegiado. Para certificación profesional, requiere upgrade. Tasa Inmuebles — Acuerdo Colegio de Arquitectos."

## Stack
- Stack JS: jsPDF, puppeteer-pdf, PDFKit, react-pdf — decisión técnica pendiente.
- Stack server-side: WeasyPrint (Python) o Playwright headless (HTML → PDF).
- Para MVP, generar **server-side** y servir como `application/pdf`.

## Validación pre-generación
- Tasación en estado ≥ Tasada.
- `valor_ARS` o `valor_USD` ≠ 0.
- Todos los datos requeridos presentes.

## Persistencia
- PDF se guarda en storage con key `tasacion-{id}.pdf` para descarga repetida sin regenerar.

## Errores
- E-001: tasación no en estado válido → 422 "Tasación no lista para PDF".
- E-002: falla del generador PDF → 500 + log para debugging.
