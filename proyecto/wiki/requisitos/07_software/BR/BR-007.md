# BR-007 — Restricción de cantidad máxima de fotos por tasación

## Enunciado

> Una [[T-026]] Tasación no admite más de **15 fotos** subidas. Cualquier intento de subir foto #16 es **rechazado** por el sistema con mensaje "Máximo 15 fotos por tasación".

## Mínimos diferenciados por tipo de tasación

- **Tasación profesional** (CU-UI-001): **mínimo 1 foto requerida**, máximo 15.
- **Tasación referencial** del Cliente B2C (CU-UI-014): **fotos opcionales** (0 admitido), máximo 5 (más bajo para reducir fricción del autoservicio).

## Parámetros configurables

| Parámetro | Valor actual | Configurable | Quién cambia |
|-----------|--------------|--------------|--------------|
| `max_fotos_tasacion_profesional` | 15 | Sí (Fase 2 vía Back Office) | Admin S-023 |
| `min_fotos_tasacion_profesional` | 1 | Sí | Admin S-023 |
| `max_fotos_tasacion_referencial` | 5 | Sí | Admin S-023 |
| `min_fotos_tasacion_referencial` | 0 | Sí | Admin S-023 |

## Reglas derivadas

- En UI (RF-006): cuando se llega a 15 fotos, el botón "+" se deshabilita con tooltip "Máximo 15 fotos".
- Validación pre-upload: si se intenta subir múltiples archivos de una vez y supera el máximo, rechazar el lote (no parcial).

## Fuente de autoridad

- Cierre de A-016 [2026-05-14, Franco]: "máximo 15 fotos por tasación".
- Justificación: balance entre riqueza visual del informe y peso del PDF + costo de storage.

## Impacto si cambia

- Aumentar a 20+: peso del PDF crece (en MVP el PDF carga todas las fotos como imágenes; en Fase 2 podría linkear a CDN).
- Reducir a 5-10: tasaciones profesionales pierden detalle visual.

## Por qué Restricción

- Formulación natural: "**no se admite** más de 15 fotos".
- El sistema **prohíbe** activamente que se supere el límite.
- Diferente de un Habilitador (que diría "se puede subir fotos solo si X").
