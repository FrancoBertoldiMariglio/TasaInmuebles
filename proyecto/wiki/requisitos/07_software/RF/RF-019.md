# RF-019 — Captura de Tasación Referencial (Cliente B2C)

## Descripción

El sistema permite al [[T-037]] **Cliente B2C** autenticado completar un formulario simplificado para generar una [[T-036]] **Tasación referencial**. **Reutiliza** la mayoría de campos de RF-005 (Detalles) y RF-007 (Descripción), con dos diferencias clave: **ubicación manual sin Google Maps** y **fotos opcionales**.

## Flujo

1. Cliente B2C en home "Mis tasaciones referenciales" toca **"+ Nueva tasación referencial"**.
2. Sistema muestra formulario:
   - **Tipo de inmueble** (chips reusados de RF-001: casa, departamento, terreno, galpón, local, oficina).
   - **Ubicación manual**:
     - `domicilio` (string libre, requerido).
     - `zona` (selector con catálogo cerrado configurable — DS-08 — para mapear a tabla `valor_m2_terreno`).
     - **Sin geolocalización ni Google Maps** (decisión 2026-05-14, MOBILE-ONLY simple).
   - **Detalles** (reusados de RF-005):
     - superficie_total_m2 (requerido).
     - superficie_cubierta_m2 (requerido).
     - dormitorios, baños (requeridos para casa/depto).
     - antigüedad_anios (requerido).
     - estado_conservacion (1-5, requerido).
     - amenities (lista, opcional).
   - **Descripción** (reusada de RF-007): texto libre, mínimo 50 caracteres.
   - **Fotos**: **opcionales** (sin requisito mínimo, hasta 5 en MVP referencial vs 15 profesional).
3. Cliente B2C toca **"Calcular valor"**.
4. Sistema valida campos requeridos.
5. Sistema persiste la tasación con:
   - `tipo_tasacion = "referencial"`.
   - `cliente_b2c_id = current_user_id`.
   - `estado = "Borrador"` inicial.
6. Sistema dispara automáticamente:
   - **RF-016** (cálculo Fitt-Servini lite) — usa el campo `zona` para lookup en tabla DS-08.
   - **Mock Robotomus** (`valor_robotomus_placeholder` = constante hardcoded MVP).
7. Sistema transiciona la tasación a estado [[T-038]] **Generada**.
8. Sistema dispara **RF-020** (generar PDF).
9. Sistema muestra pantalla de resultado:
   - Valor técnico Fitt-Servini (ARS / USD) con desglose.
   - Valor referencial Robotomus (ARS / USD — etiquetado "placeholder, MVP").
   - Botones "Descargar PDF" y "Enviar a mi email".

## Validación

- Mismas reglas de validación de RF-005 (superficies positivas, dormitorios/baños según tipo, etc.).
- `zona` ∈ catálogo configurable (DS-08).
- Mínimo descripción 50 caracteres (heredado de RF-007).
- Fotos: max 5 archivos, max tamaño cada una a definir AC.

## UI MVP

- Mismo formulario "acordeón" que CU-UI-001 (consistencia con la app del tasador).
- **Sin** botón "Enviar a comité" (las referenciales no van a comité).
- **Sin** botón "Inspección ocular" (Fase 2 si se agrega).
- CTA principal: "Calcular valor" en lugar de "Guardar cambios".

## Diferencias clave vs RF-008 (Guardar tasación profesional)

| Aspecto | RF-008 Profesional | RF-019 Referencial |
|---|---|---|
| Geolocalización | RF-003 (HTML5 + Google Maps) | **Manual** (string + zona) |
| Fotos mínimas | ≥ 1, máx 15 | Opcionales, máx 5 |
| Estado al guardar | Inspección hecha → Por tasar | Borrador → Generada (auto-cálculo) |
| Va a comité | Sí (CU-UI-004) | No (es auto) |
| Valor final | Cerrado por comité (RF-015) | Generado automáticamente (RF-016 + mock) |
| PDF generado | RF-012 (PDF profesional, firma plataforma o colegiado) | RF-020 (PDF referencial con disclaimer "no certificada") |
