# RF-003 — Geolocalizar inmueble

## Descripción
El sistema captura `(latitud, longitud, domicilio)` del inmueble usando uno de dos modos:

## Sub-flujo A — "Usar mi ubicación actual"
1. Tasador toca botón "Usar mi ubicación actual".
2. Sistema invoca HTML5 Geolocation API.
3. Browser/OS pide permiso al usuario.
4. Si permiso concedido: sistema recibe `(lat, lng)` con precisión.
5. Sistema invoca reverse geocoding de Google Maps API para obtener `domicilio` (string formateado).
6. Sistema persiste `(lat, lng, domicilio, modo_carga='ubicacion_actual')` en draft.

## Sub-flujo B — Ingreso manual con Google Maps Places
1. Tasador toca campo "Domicilio".
2. Sistema muestra input con autocomplete de Google Maps Places API.
3. Tasador escribe parcialmente y selecciona del dropdown.
4. Sistema recibe `place_id` + `domicilio` + `(lat, lng)`.
5. Sistema persiste `(lat, lng, domicilio, modo_carga='manual_google_maps')` en draft.

## Errores
- E-001: HTML5 Geolocation denegado o sin señal → mostrar mensaje y derivar a sub-flujo B.
- E-002: Google Maps API no responde → guardar `domicilio` como string libre + lat/lng vacíos. Notificar al tasador.

## Integración
- Google Maps Places API + Google Maps Geocoding API.
- HTML5 Geolocation (browser/Capacitor/React Native dependiendo del stack final).

## Validación
- Para guardar (RF-008), se requiere al menos uno de: `(lat, lng)` o `domicilio` no vacío.
