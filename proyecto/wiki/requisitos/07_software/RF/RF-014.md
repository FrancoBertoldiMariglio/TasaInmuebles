# RF-014 — Enviar PDF por mail al solicitante (opción C de "Compartir")

## Descripción
El sistema envía un mail pre-diseñado al [[T-032]] Solicitante con el PDF de la [[T-026]] adjunto.

## Flujo
1. Tasador en la ficha (estado ≥ Tasada) toca "Enviar por mail al solicitante".
2. Sistema valida: solicitante tiene email registrado (RF-004). Si no: botón deshabilitado con tooltip.
3. Sistema muestra preview del mail:
   - **De:** `tasaciones@tasacioninmuebles.com` (o similar).
   - **Para:** `solicitante.email`.
   - **Asunto:** `Tasación de {domicilio} — Tasa Inmuebles`.
   - **Cuerpo:** template predefinido (texto + firma).
   - **Adjunto:** PDF generado por RF-012.
4. Tasador puede editar el cuerpo (opcional MVP).
5. Tasador toca "Enviar".
6. Sistema envía via proveedor SMTP (DS-07: Resend / Postmark / Mailgun).
7. Sistema persiste registro: `email_event(tasacion_id, destinatario, timestamp, provider_message_id, status)`.
8. Sistema muestra confirmación "Mail enviado a {email}".
9. Sistema transiciona la tasación a [[T-007]] Compartida.

## Errores
- E-001: solicitante sin email → botón deshabilitado.
- E-002: proveedor SMTP rechaza → mostrar error específico + log.
- E-003: bounce o spam-block del destinatario → notificar al tasador.

## Integración
- Proveedor SMTP a elegir entre Resend / Postmark / Mailgun (DS-07). Sugerencia: Resend por simplicidad de API.

## Seguridad
- Solo `tasador_id` autor o miembros del comité pueden enviar.
- Rate limit: ≤ 5 envíos por tasación por día (anti-abuse).

## Fuera del MVP-6sem
- Link único auditable con token (opción B descartada en A-016).
- Tracking de apertura del mail (pixel transparente).
- WhatsApp Business API.
