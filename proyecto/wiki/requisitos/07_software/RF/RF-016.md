# RF-016 — Calcular valor técnico vía fórmula Fitt-Servini

## Descripción

El sistema calcula automáticamente el [[T-030]] **Valor técnico** de una [[T-026]] aplicando la fórmula determinística de [[T-011]] **Fitt y Servini**. El cálculo se dispara **al guardar la tasación** (después de RF-008, antes de transicionar a [[T-022]]) y persiste el resultado.

## Trigger

- Automático al pasar de [[T-002]] Borrador → [[T-014]] Inspección hecha (consecuencia de RF-008 paso 4).
- Re-cálculo automático si el tasador edita campos relevantes de "Detalles" (RF-005) antes del cierre del comité (RF-015).

## Inputs (provienen del modelo de Tasación, RF-005)

| Campo | Origen | Notas |
|-------|--------|-------|
| `tipo_inmueble` | RF-001 | Determina si aplica superficie_cubierta y/o terreno |
| `superficie_total_m2` | RF-005 | |
| `superficie_cubierta_m2` | RF-005 | 0 para terrenos puros |
| `antiguedad_anios` | RF-005 | |
| `estado_conservacion` (1-5) | RF-005 | Determina `coef_conservacion` (BR-006) |
| `latitud`, `longitud` | RF-003 | Determinan **zona** (lookup contra tabla) |

## Fórmula Fitt-Servini lite (MVP-6sem)

```
superficie_terreno = max(superficie_total_m2 - superficie_cubierta_m2, 0)

valor_construccion = superficie_cubierta_m2
                   × valor_m2_categoria[tipo_inmueble][categoría]
                   × coef_conservacion[estado_conservacion]

valor_terreno = superficie_terreno
              × valor_m2_terreno[zona]
              × coef_frente_fondo    // = 1.0 en MVP, refinar Fase 2

valor_tecnico_USD = valor_construccion + valor_terreno
valor_tecnico_ARS = valor_tecnico_USD × tipo_cambio_referencia
```

### Parámetros (tabla configurable)

**`coef_conservacion`** (escala 1-5, ya definida en BR-006):

| estado_conservacion | Etiqueta | Coeficiente |
|---|---|---|
| 5 | A estrenar | 1.00 |
| 4 | Muy bueno | 0.85 |
| 3 | Bueno | 0.70 |
| 2 | Regular | 0.55 |
| 1 | A refaccionar | 0.40 |

**`valor_m2_categoria`** y **`valor_m2_terreno`** (a parametrizar):

- En **MVP-6sem**: tabla hardcodeada en archivo `config/fitt_servini_table.json`, valores cargados por Franco junto con un arquitecto del Colegio.
- En **Fase 2**: módulo admin que permite a un owner / arquitecto autorizado actualizar la tabla por zona, tipo, categoría.

**Decisión pendiente DS-08**: definir la tabla inicial junto con un arquitecto del Colegio antes del Hito 1. Bloqueante para que el cálculo dé números realistas.

**`tipo_cambio_referencia`** (ARS/USD):

- MVP-6sem: valor manual cargado por admin (1 por mes o cuando el dólar varíe significativamente).
- Fase 2: API externa (BCRA, dolarapi.com, similar).

## Outputs

| Campo persistido | Tipo | Notas |
|------------------|------|-------|
| `valor_tecnico_USD` | money | Resultado de la fórmula. |
| `valor_tecnico_ARS` | money | Con tipo de cambio aplicado. |
| `valor_tecnico_inputs` | JSON | Snapshot de los inputs usados para el cálculo (auditoría). |
| `valor_tecnico_fecha_calculo` | timestamp | Cuándo se calculó (cambia si hay re-cálculo). |
| `valor_tecnico_version_formula` | string | "fitt-servini-lite-v1" — para versionar la fórmula a futuro. |

## Visibilidad

- En la ficha de la tasación (vista de consulta) y en CU-UI-004 (comité), el valor técnico se muestra **calculado y editable**:
  - Si el comité **acepta** el valor calculado, lo confirma con el botón "Usar como valor final" → llena automáticamente `valor_final_ARS` y `valor_final_USD`.
  - Si el comité **discrepa** (por algún factor que la fórmula no captura), puede sobrescribirlo manualmente. La sobrescritura queda registrada como `override_motivo` (texto libre opcional).

## Validación

- `valor_tecnico_USD > 0` y `valor_tecnico_ARS > 0` (si el resultado da 0 o negativo, error de cálculo — log + alert al admin).
- Si falta algún input requerido, se persiste `valor_tecnico = null` y se muestra al comité "No se pudo calcular: faltan datos. Completar [campos] y reintentar."
- Si la zona del inmueble no está en la tabla `valor_m2_terreno`, se aplica el valor de "zona desconocida" (fallback configurable) y se flagea.

## Diferenciador para el Colegio

Mostrar **valor técnico Fitt-Servini calculado automáticamente** es el diferenciador clave para los arquitectos del Colegio: en lugar de calcularlo a mano cada vez, lo ven listo al abrir el comité. Es coherente con el espíritu profesional del producto (no es "magia ML" — es la fórmula que ellos mismos enseñan).

## Fuera del MVP-6sem

- Coeficiente frente/fondo dinámico calculado desde la geometría del terreno (Fase 2).
- Coeficientes de antigüedad refinados (depreciación lineal vs exponencial, Fase 2).
- Tabla `valor_m2_*` administrable vía UI (Fase 2).
- Tipo de cambio vía API automática (Fase 2).
- Versionado de fórmula (Fase 3+ si los arquitectos proponen mejoras).
