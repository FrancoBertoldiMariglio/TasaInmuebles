# RF-005 — Capturar detalles del inmueble

## Descripción
El sistema captura los detalles físicos del inmueble necesarios para tasación + entrada futura de [[T-023]].

## Campos
| Campo | Tipo | Aplica a | Requerido |
|-------|------|----------|-----------|
| `superficie_total_m2` | number positivo | todos los tipos | sí |
| `superficie_cubierta_m2` | number positivo | todos | sí (puede ser 0 para terreno) |
| `dormitorios` | int ≥ 0 | casa, departamento | sí (no aplica a otros) |
| `banos` | int ≥ 0 | casa, departamento | sí (no aplica a otros) |
| `antiguedad_anios` | int ≥ 0 | todos | sí |
| `estado_conservacion` | int 1-5 (escala ordinal) | todos | sí |
| `amenities` | array de strings (lista cerrada Q12) | todos | opcional |

## Escala estado_conservacion (BR-006)
1 = A refaccionar · 2 = Regular · 3 = Bueno · 4 = Muy bueno · 5 = A estrenar
(Coeficientes Fitt-Servini: 0.40 / 0.55 / 0.70 / 0.85 / 1.00)

## Validación
- Superficies ≥ 0 (números reales positivos).
- Dormitorios/baños: enteros ≥ 0.
- `superficie_cubierta_m2 ≤ superficie_total_m2`.
- estado_conservacion ∈ {1, 2, 3, 4, 5}.

## UI
- Acordeón "Detalles" en formulario.
- Estado de conservación: slider o radio buttons con etiquetas.
- Amenities: checkboxes según Q12 (pendiente lista canónica).
