# RF-010 — Login con email + password

## Descripción
El sistema autentica al [[T-028]] con email + password contra base de datos local. Genera JWT estándar.

## Flujo
1. Tasador ingresa email y password en pantalla de login.
2. Sistema busca usuario por email en BD.
3. Sistema verifica password (hash bcrypt o argon2).
4. Si OK: genera JWT con claim `sub = user_id` + claim `role = tasador`.
5. Sistema retorna JWT al cliente.
6. Cliente persiste JWT en localStorage / secure storage.
7. Cliente redirige al dashboard.

## Errores
- E-001: email no existe → mensaje genérico "Email o contraseña incorrectos".
- E-002: password incorrecta → mismo mensaje (no diferenciar para evitar enumeration).
- E-003: usuario inactivo → mensaje "Cuenta deshabilitada. Contactá al admin."

## Seguridad
- Password se guarda hasheado (nunca plaintext).
- JWT con expiración (sugerencia: 8 horas para MVP).
- Sin refresh token en MVP (re-login al expirar).
- RG-003 (TLS) aplica.

## Fuera del MVP-6sem
- Auth0 / Clerk / Supabase Auth.
- Google OAuth, SSO empresarial.
- Recuperación de contraseña.
- MFA.
