# RF-020 — Generar PDF de Tasación Referencial

## Descripción

El sistema genera un PDF para una [[T-036]] **Tasación referencial** en estado [[T-038]] **Generada**, con disclaimer fuerte de "no certificada profesionalmente".

## Contenido del PDF (diferencias vs RF-012 profesional)

| Sección | Profesional (RF-012) | Referencial (RF-020) |
|---------|------|------|
| Encabezado | "Tasación N° X" | **"Tasación Referencial N° X"** (badge visible) |
| Disclaimer | Footer pequeño | **Banda destacada en primera página** con texto fuerte |
| Datos del inmueble | Completos + geolocalización con mapa | Completos + dirección manual (sin mapa) |
| Fotos | Foto principal grande + grid | Si hay fotos: grid simple; si no hay: sección vacía |
| Valor final | Cerrado por comité | **Valor técnico Fitt-Servini** + Valor referencial Robotomus mock |
| Firma | "Tasa Inmuebles" + IDs comité + tasador | **"Generado automáticamente. Sin firma profesional."** |
| Cliente | Solicitante (RF-004) | El propio Cliente B2C |
| Footer | "Para certificación, requiere upgrade" | **"Esta tasación es REFERENCIAL. NO es válida para operaciones legales (hipoteca, sucesión, judicial). Para certificación profesional, consulte con un tasador colegiado."** |

## Disclaimer obligatorio (BR-021 pendiente)

Texto literal recomendado, a validar con Cocucci (responsabilidad legal del Colegio):

> ⚠️ **TASACIÓN REFERENCIAL — NO CERTIFICADA PROFESIONALMENTE**
>
> Esta tasación fue generada automáticamente por Tasa Inmuebles aplicando fórmulas estándar (Fitt-Servini) sobre los datos provistos por el usuario, sin inspección ocular ni revisión profesional.
>
> **NO es válida** para: hipotecas, sucesiones, expropiaciones, operaciones judiciales, garantías legales.
>
> Para una **tasación certificada con firma profesional**, contacte un tasador colegiado matriculado.

## Flujo

1. RF-019 completa exitosamente → tasación en estado [[T-038]] Generada.
2. Sistema dispara RF-020.
3. Sistema renderiza template HTML del PDF referencial (distinto al profesional).
4. Sistema persiste el PDF en storage con key `tasacion-ref-{id}.pdf`.
5. Sistema marca `pdf_generado = true` + `fecha_generacion_pdf = now()`.
6. Sistema notifica al cliente que el PDF está disponible.

## Stack
- Mismo motor que RF-012 (jsPDF / WeasyPrint / Playwright headless — DP-007).
- Template HTML separado para referencial (distinto al profesional).

## Distribución (Compartir, sigue lógica de CU-UI-005 adaptada)

- **Descarga directa**: botón "Descargar PDF" — siempre disponible.
- **Envío a email**: opción de enviar el PDF al email registrado del cliente (mismo email del autoregistro, RF-018). Reusa RF-014 (Enviar PDF por mail) parametrizando el cliente como destinatario.

## Auditoría

- Persistir `pdf_generated_event(tasacion_id, cliente_b2c_id, timestamp, version_pdf="referencial-v1")`.
- Permite reportar volumen de referenciales generadas para métrica de éxito MVP.

## Compliance

- Disclaimer es **obligatorio** (BR-021) — no se puede generar PDF sin él.
- En Fase 2 se versiona la fórmula → cambia el badge en el PDF a "referencial-v2" etc.
